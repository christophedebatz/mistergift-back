AUTHENTICATION
Description: when user is asking for log in

- Request:
POST /authenticate
Body enctype: x-www-form-urlencoded
Body content: email=<user-email>&password=<user-raw-password>

- Response:
Body success content:
{
  "status": 200,
  "payload": {
    "id": <user-id>,
    "email": "<user-email>",
    "firstName": "<user-first-name>",
    "lastName": "<user-last-name>",
    "session": {
      "token": "<token-value>",
      "expireAt": <expire-at-timestamp>
    }
  }
}

Body failure content:
{
  "error": {
    "status": 403,
    "exception": "<exception-type>",
    "message": "<exception-message>"
  }
}

AUTHORIZATION

Request:
[GET|POST|PUT|DELETE] /resource
So, you must pass a valid token in header named "X-MG-AUTH" to access to user-only resources.
